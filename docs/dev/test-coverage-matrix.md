# Parsek Test Coverage Matrix

Date: 2026-04-21

This is a current-tree subsystem matrix, not a percentage report. It complements [test-coverage-audit-2026-04-19.md](test-coverage-audit-2026-04-19.md) by mapping the major production areas in `Source/Parsek` to the four coverage styles the repo actually uses today:

- headless xUnit tests in `Source/Parsek.Tests`
- live runtime checks in `Source/Parsek/InGameTests`
- `KSP.log` contract validation via [`scripts/validate-ksp-log.ps1`](../../scripts/validate-ksp-log.ps1)
- manual scenario checklists in [manual-testing/test-general.md](manual-testing/test-general.md) and [manual-testing/test-auto-record.md](manual-testing/test-auto-record.md)

Rows are intentionally tied to named source owners that exist in the current tree, not planned future seams.

## Legend

- `Strong`: direct named owners are easy to find in the current tree.
- `Targeted`: direct coverage exists, but it is narrower, scenario-specific, or incomplete for the whole subsystem.
- `Indirect`: the subsystem benefits from neighboring tests or generic contracts, but does not have a clear dedicated owner in that coverage mode.
- `None`: no obvious current-tree evidence.

## Cross-Cutting Gaps

- There is still no mechanical line/branch coverage report in the repo, so this matrix is intentionally qualitative.
- The strongest automation is concentrated in extracted logic, persistence, playback seams, and diagnostics rather than UI shell plumbing.
- Runtime coverage is broad, but many of the highest-value player journeys are still assembled from invariant checks plus manual playtests instead of one reproducible scenario script.
- Manual checklists are high quality and aligned with real product risk, but they remain human-enforced evidence.
